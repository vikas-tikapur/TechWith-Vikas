Common OOP Questions in Python

Enhance your understanding of Object-Oriented Programming in Python with these commonly asked and important questions every developer should know.



A class is a blueprint or template, while an object is an instance of that class. The class defines structure and behavior; the object holds actual data and performs actions.

Instance variables are specific to each object and stored in the object’s memory.

Class variables are shared across all objects and defined at the class level.

A constructor is a special method used to initialize objects when a class is instantiated. In Python, it is defined using the __init__( ) method.

The __init__ method is a special constructor in Python that is automatically called when a new object is created. It is used to initialize the object's attributes.

self refers to the current object (instance) of the class and is used to access instance variables and methods.

Instance methods use self and access instance data.

Class methods use @classmethod and cls to access class data.

Static methods use @staticmethod and don’t access instance or class data.

Getters retrieve values, and setters update them. They are defined using regular methods or the @property decorator.

Single inheritance: One child class inherits from one parent.

Multiple inheritance: One child inherits from multiple parents.

When a subclass provides its own implementation of a method already defined in the parent class.

super( ) is used to call methods or constructors from the parent class inside a child class.

Operator overloading allows custom behavior for operators using magic methods like __add__, __eq__, etc.

Method overloading means having multiple methods with the same name but different parameters.
Python doesn't support it directly; only the last defined method is used.

Abstract classes can't be instantiated and must be subclassed. Use ABC module and @abstractmethod to define them.

  • @classmethod accesses class-level data (cls)
  • @staticmethod has no access to class or instance
  • Instance methods access object data using self

__str__( ) returns a user-friendly string.

__repr__( ) returns a developer-friendly string (for debugging).

Using single _ (protected) and double __ (private) underscores before variable names.

Special methods with double underscores, like __init__, __add__, __str__, that add functionality to classes.

__dict__ returns a dictionary of an object’s attributes.

Yes, a class can be defined inside another class.

Example :

class Outer:
    class Inner:

Instance variables are stored separately for each object; class variables are shared and stored in class memory.

Python uses reference counting and a garbage collector to automatically clean up unused objects.

MRO defines the order in which methods are searched in a hierarchy. Python uses the C3 linearization algorithm.

Inheritance: "Is-a" relationship

Composition: "Has-a" relationship (using one class inside another)

Yes, a Python function can return multiple values by separating them with commas. Internally, they are returned as a tuple.

Example :

def get_student():
    name = "Rishik"
    age = 22
    marks = 85
    return name, age, marks

student_info = get_student()
print(student_info)  # Output: ('Rishik', 22, 85)

A Class Attribute is shared by all objects of the class.

An Instance Attribute is unique to each object.

Scope refers to the region or area in a program where a variable is recognized or accessible.


Types of Scope in Python (LEGB Rule) :

1. Local - Inside a function.

2. Enclosed - Inside a nested function.

3. Global - At the top-level of the script

4. Built-in - Python’s reserved names like print( ), len( ), etc.
